Local-first Agentic RAG for personal folders

Find files by memory, not filename.

Browhere indexes only approved local folders, embeds text, image, OCR, and metadata evidence, then retrieves and answers with bounded provider calls and local citations.

$ search "finance pdf from Reports folder"
metadatabudget.pdf92.4%
textquarterly-risk-notes.md84.8%
$ answer "what changed in launch notes?"
Answer uses retrieved evidence only. Citations map to file path and evidence id.

Problem

Desktop search works when you remember exact strings. Browhere is for the messy cases: vague filenames, image-only clues, PDF/DOCX contents, and remembered concepts.

01

Literal Search Breaks

Files named IMG_4021.jpg or notes-final-v3.md are hard to find with keyword-only search.

02

Useful Signals Hide

The clue might be a logo in an image, text in a PDF, a parent folder, or the general topic of a document.

03

Privacy Matters

The index stays local. Gemini and Groq receive only the bounded content needed for embedding, retrieval, reranking, or answering.

Approve a folder, let Browhere index supported files, then search by concept or ask a grounded question.

Agentic RAG Method

Traditional RAG retrieves top-k chunks and answers. Browhere adds query understanding, bounded planning, hybrid ranking, evidence provenance, and answer citations.

Diagram 03Retrieval and Answer Pipeline
Indexing Lane
  1. Discover approved files and ignore excluded paths.
  2. Extract text from TXT, MD, PDF, and DOCX.
  3. Embed raw images and generate visual-caption evidence.
  4. Extract OCR text from images when readable text exists.
  5. Store vectors, chunks, metadata, provenance, and repair status locally.
Retrieval Lane
  1. Interpret query intent: file type, folder hint, visual intent, OCR intent, answer intent.
  2. Run semantic retrieval with configurable top-k and retrieval passes.
  3. Add lexical, filename/path, and metadata candidates.
  4. Apply hybrid score components and source caps.
  5. Optionally ask Groq to rerank bounded candidates.
  6. Generate answer only from selected evidence and cite local evidence ids.

Traditional RAG

  • Single query embedding.
  • Vector top-k retrieval.
  • Context pack sent to answer model.
  • Citations often point only to source documents.

Browhere Agentic RAG

  • Query planning and structured intent.
  • Semantic, lexical, metadata, OCR, and visual evidence.
  • Hybrid scoring with diagnostics and source caps.
  • Answer citations map to file path, evidence id, provenance, and location metadata.

Quality Gates

Tests are mapped to RAG failure modes: bad indexing, weak retrieval, provider outage, uncited answers, and UI regressions.

Test Case Build

  • Discovery and exclusion tests prevent unsafe indexing.
  • Extraction tests verify chunk creation and partial fallbacks.
  • Repository tests verify LanceDB persistence and compatibility normalization.
  • Indexer tests verify raw image, visual caption, OCR, metadata, provider failure, and repair tasks.
  • Retrieval eval fixtures verify expected top file, expected evidence source, citation behavior, and insufficient evidence.

Representative Queries

  • document about reptiles in warm habitats
  • finance pdf from Reports folder
  • what words are visible on the menu screenshot
  • Answer mode with valid local evidence citations.
  • Answer mode with invalid citations returns insufficient evidence.

Setup

Gemini is required for embeddings. Groq is optional for agentic planning, reranking, and answer generation.

Required

  • Node.js 20 or newer.
  • GEMINI_API_KEY for indexing and semantic search.
  • Approved local folder path from the UI.

Optional

  • GROQ_API_KEY for planning, reranking, explanations, answers.
  • HUGGINGFACE_API_KEY for alternate image caption provider.
  • BROWHERE_INDEX_DIR to move the local index.
Browhere is not a remote search product. It is a local evidence machine with bounded cloud reasoning.
